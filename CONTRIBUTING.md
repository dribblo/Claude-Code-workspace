# Contributing to claude-context

`claude-context` is a community standard. It gets better when more people bring their workflows to it.

## What we're looking for

### Branch templates

The most valuable contributions are branch templates for specific roles and professions:

* `lawyer.md` — legal research, document drafting, client communication
* `designer.md` — design critique, brief writing, visual direction
* `teacher.md` — lesson planning, student communication, curriculum
* `founder.md` — investor comms, hiring, product decisions
* `consultant.md` — client communication, deliverables, engagement management
* `researcher.md` — literature review, methodology, academic writing
* `chief-of-staff.md` — exec support, cross-functional coordination, ops

A good template is specific enough to be immediately useful, and generic enough to work for most people in that role.

### Spec improvements

If the spec is unclear, incomplete, or wrong — open an issue. Changes require discussion before a PR.

### Translations

README and SPEC in other languages are welcome.

## How to contribute a branch template

1. Fork the repo
2. Copy `branches/project-template.md` to `branches/{role}.md`
3. Fill in all 5 sections with realistic, useful content
4. Remove all placeholder text
5. Open a PR with a one-line description of who this is for

## How to propose a spec change

1. Open an issue describing the problem
2. Wait for discussion
3. PR with updated `SPEC.md` + version bump

## Code of conduct

Be direct. Be constructive. Contributions are judged on whether they make the standard more useful for real people.
